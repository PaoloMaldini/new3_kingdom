# 《楼台烟雨》游戏设计文档

**版本：** 1.0
**日期：** 2026年4月
**状态：** 初稿

---

## 一、游戏简介

### 1.1 游戏概述

《楼台烟雨》是一款以中华古代建筑为主题的水墨风互动体验游戏。玩家通过水墨风格的交互式地图，探索1911年前的中国古代建筑（民居、官府、皇宫、桥梁），点击建筑标记进入历史场景，体验建筑背后的故事与人物命运。

### 1.2 作品主题

**大赛主题：** 中国古代建筑成就——中华优秀传统文化系列之六

**内容范围：**
- 民居建筑（延安窑洞、福建土楼、徽州民居、北京四合院、川西民居、江南水乡民居）
- 官府建筑（天一阁、包拯祠、黄鹤楼）
- 皇宫建筑（故宫、阿房宫、大明宫）
- 桥梁建筑（赵州桥、卢沟桥、断桥、宝带桥）

**时间限制：** 1911年以前

### 1.3 游戏特色

- **水墨美学**：首创水墨风格古建筑地图，还原中国山水画的淡雅意境
- **沉浸式剧本**：每个建筑配备专属历史剧本，玩家以"亲历者"身份进入场景
- **交互式叙事**：左右滑动选择，实时反馈数值变化，体验选择对命运的影响
- **双模式并行**：保留三国地图模式，提供两种不同的游戏体验

## 二、创作立意

### 2.1 创作背景

中国古代建筑是中华优秀传统文化的重要组成部分，承载着数千年的历史记忆与工匠智慧。然而，当代年轻人对古建筑的了解往往局限于走马观花的观光，缺乏深入的文化体验。

### 2.2 创作理念

**"让古建筑活起来"** —— 不同于传统的图文介绍或视频展示，本作采用"互动叙事"的方式，让玩家真正"进入"历史场景，以第一人称视角体验古建筑背后的人物故事，在潜移默化中感受中华建筑的独特魅力。

### 2.3 文化传承目标

1. **知识传递**：通过建筑数据与历史故事，传递古建筑的基本知识
2. **审美培养**：以水墨美学激发玩家对传统审美的兴趣
3. **情感共鸣**：通过沉浸式叙事，让玩家与历史人物产生情感连接
4. **文化认同**：在娱乐中增强对中华优秀传统文化的认同感

### 2.4 创新定位

| 传统方式 | 本作创新 |
|---------|---------|
| 静态图文介绍 | 水墨风动态地图 |
| 单向信息输出 | 双向互动体验 |
| 旁观者视角 | 亲历者视角 |
| 知识记忆 | 情感共鸣 |

## 三、核心玩法

### 3.1 游戏模式

#### 模式一：古建筑探索（主模式）

```
首页 → 古建筑地图 → 选择建筑 → 建筑详情页 → 剧本互动 → 结束
```

**流程说明：**
1. 玩家进入水墨风地图，浏览中国古代建筑分布
2. 可按分类（民居/官府/皇宫/桥梁）筛选建筑
3. 点击建筑标记，查看建筑详情（图片、历史、人物）
4. 阅读剧本简介，点击"开始体验"进入互动剧本
5. 通过左右滑动做出选择，影响角色属性
6. 完成剧本后查看最终成绩，可选择重玩或返回地图

#### 模式二：三国地图（保留模式）

```
首页 → 三国地图 → 选择主公 → 选择地点 → 剧本互动 → 结束
```

**流程说明：**
1. 玩家选择一位三国主公（曹操、刘备、孙坚、袁绍、董卓）
2. 在地图上选择地点，进入该地点的剧本
3. 通过左右滑动做出选择，体验主公的人生历程
4. 根据属性变化（天意/权谋/兵力/粮草）触发不同结局

### 3.2 属性系统

游戏中设有四项核心属性，通过选择影响其数值变化：

| 属性 | 含义 | 最高值 | 最低值 |
|-----|------|-------|-------|
| 天意 | 命运、时机、民意 | 100 | 0 |
| 权谋 | 政治智慧、策略 | 100 | 0 |
| 兵力 | 军事力量、武装 | 100 | 0 |
| 粮草 | 资源储备、经济 | 100 | 0 |

**结局触发条件：**
- 任意属性达到100：触发"极致"结局
- 任意属性降至0：触发"耗尽"结局

### 3.3 剧本结构

每个建筑的剧本包含2个场景事件，每个事件提供2个选择：

```
建筑剧本
├── 事件1
│   ├── 选择A → 属性变化 → 事件2
│   └── 选择B → 属性变化 → 事件2
└── 事件2（最终事件）
    ├── 选择A → 属性变化 → 结局
    └── 选择B → 属性变化 → 结局
```

### 3.4 交互流程

```
[显示事件卡片]
       ↓
[玩家左右滑动/点击选择]
       ↓
[实时预览属性变化动画]
       ↓
[滑动超过阈值 → 确认选择]
       ↓
[显示属性变化反馈]
       ↓
[进入下一事件或结局]
```

```typescript
// 滑动阈值常量
const SWIPE_THRESHOLD = 80  // 触发确认的最小滑动距离
const DRAG_THRESHOLD = 30   // 开始响应滑动的最小距离

// 事件处理
function handlePointerDown(e: PointerEvent) {
  isDragging.value = true
  dragStartX.value = e.clientX
  document.addEventListener('pointermove', handlePointerMove)
  document.addEventListener('pointerup', handlePointerUp)
}

function handlePointerMove(e: PointerEvent) {
  if (!isDragging.value) return
  dragDistance.value = e.clientX - dragStartX.value
  
  if (dragDistance.value < -DRAG_THRESHOLD) {
    dragDirection.value = 'left'
  } else if (dragDistance.value > DRAG_THRESHOLD) {
    dragDirection.value = 'right'
  }
}

function handlePointerUp() {
  if (Math.abs(dragDistance.value) >= SWIPE_THRESHOLD) {
    applyChoice(dragDirection.value)
  }
  // 重置状态...
}
```

## 六、重难点说明

### 6.1 技术重点

#### 水墨地图渲染

**问题描述：** 如何在Web端实现水墨画的视觉效果，包括山脉轮廓、墨点纹理、渐变晕染等。

**解决方案：**
- 使用SVG路径绘制简化中国地图轮廓
- 使用CSS渐变实现远山渐隐效果
- 使用SVG滤镜（feTurbulence、feDisplacementMap）实现墨点扩散
- 使用伪元素叠加毛笔笔触装饰

**实现代码示例：**

```css
/* 水墨山峦效果 */
.mountain {
  filter: url(#inkBlur);
  opacity: 0.6;
}

/* 墨点纹理叠加 */
.ink-texture {
  background:
    radial-gradient(ellipse at 20% 30%, rgba(30, 30, 40, 0.4) 0%, transparent 50%),
    radial-gradient(ellipse at 80% 70%, rgba(25, 25, 35, 0.3) 0%, transparent 40%);
}
```

#### 滑动交互优化

**问题描述：** 在桌面端和移动端都需要流畅的滑动体验，且需要支持触摸和鼠标两种交互方式。

**解决方案：**
- 使用Pointer Events API统一处理触摸和鼠标事件
- 使用`touch-action: none`禁用浏览器默认手势
- 添加CSS transform实现实时视觉反馈
- 设置合理的阈值防止误触

### 6.2 内容重点

#### 古建筑数据收集

**问题描述：** 需要收集符合大赛要求的古建筑数据，包括建筑简介、历史意义、关联人物、适合剧本化的历史场景。

**解决方案：**
- 严格遵循大赛要求：民居、官府、皇宫、桥梁四类
- 每个建筑配备：基本信息、3-5个关联人物、2个可剧本化的事件
- 事件设计围绕建筑相关的真实历史故事
- 选择结果影响虚构属性（非历史事实）

#### 剧本设计

**问题描述：** 如何在尊重历史的前提下，设计有趣的互动选择。

**解决方案：**
- 剧本设定为玩家的"亲历视角"，而非改变历史
- 选择描述侧重于人物的心理活动和处世态度
- 属性变化体现选择的"后果"而非"对错"
- 提供多样化的结局，而非单一的历史正确路线

### 6.3 性能重点

#### 首屏加载优化

**问题描述：** 水墨风格地图包含大量SVG元素和装饰，可能影响首屏加载速度。

**解决方案：**
- 使用CSS动画实现淡入效果，分散加载感知
- 图片资源使用CDN托管，减少本地体积
- 按需加载建筑详情数据
- 使用Vite进行代码分割

#### 响应式适配

**问题描述：** 游戏需要在桌面端和移动端都有良好的体验。

**解决方案：**
- 使用CSS clamp()和dvh/dvw实现流式布局
- 地图交互在移动端通过滑动，桌面端通过鼠标拖拽
- 侧边栏在移动端隐藏，改为点击查看详情
- 断点设计：900px（平板）、640px（手机）

### 6.4 兼容性重点

#### 浏览器兼容

| 浏览器 | 最低版本 | 备注 |

| Chrome | 90+ | 推荐 |
| Firefox | 88+ | 支持 |
| Safari | 14+ | 支持 |
